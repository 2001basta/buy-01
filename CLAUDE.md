# Marketplace Microservices — Project Context

Spring Boot 3 microservices backend + Angular 17 frontend.
Sellers can list products with images. Clients browse products.
Six services total: Config, Discovery, User, Product, Media, API Gateway.

---

## Architecture

```
Angular → API Gateway (8080)
              ↓  JWT validation + Correlation-ID injection
              ↓  lb:// URIs resolved via Eureka
   ┌──────────┬──────────┬──────────┐
   User(8081) Product(8082) Media(8083)
        │           │            │
        └───────────┴────────────┘
               ↕ Kafka (product-events topic)
        [MongoDB per service — never shared]

Infrastructure:
  Config Server (8888)    ← centralises shared + per-service config
  Discovery / Eureka (8761) ← service registry, lb:// routing
  Zipkin (9411)           ← distributed tracing
  Kafka / KRaft (9092)    ← async events, no Zookeeper
```

Each service has its own MongoDB database. Do NOT query across databases.

## Tech Stack

- **Backend:** Spring Boot 3.5, Spring Security 6, Spring Cloud Gateway (reactive/webflux)
- **Service Discovery:** Spring Cloud Netflix Eureka
- **Config:** Spring Cloud Config Server (native profile, classpath config files)
- **Database:** MongoDB (one database per service, never shared)
- **Events:** Apache Kafka (KRaft mode, no Zookeeper) — topic `product-events`
- **Resilience:** Resilience4j circuit breaker on ProductService → MediaService call
- **Tracing:** Micrometer Tracing + Brave bridge + Zipkin
- **Frontend:** Angular 17+, standalone components, typed HttpClient, JWT interceptor
- **Auth:** JWT (jjwt 0.12.6) — stateless, no sessions. BCrypt for passwords.

## Critical Business Rules — Always Enforce

1. **Only SELLER role can create/update/delete products.**
   Check `roles.contains("SELLER")` before any write. Return `403 FORBIDDEN` otherwise.

2. **Only the product owner can modify it.**
   Check `product.sellerId == currentUserId`. Never skip this.

3. **File uploads: 2MB max, image/jpeg or image/png only.**
   Validate with Apache Tika (magic bytes) — never trust `file.getContentType()`.

4. **Only the media owner can delete their file.**
   Check `media.ownerId == currentUserId`.

5. **Ownership checks go in the service layer, not the controller.**

## JWT

Token payload: `{ sub: userId, roles: ["SELLER"|"CLIENT"], exp: <unix> }`

- Generated by UserService using jjwt 0.12.6 (`Keys.hmacShaKeyFor`, HS256)
- Validated at the **Gateway** (`JwtAuthFilter`) before forwarding
- Gateway injects `userId` and `roles` as request headers to downstream services
- Services read `@RequestHeader("userId")` and `@RequestHeader("roles")` — never re-parse the JWT
- Password is **never** returned in any response
- JWT secret must be **at least 32 characters** (jjwt 0.12.6 enforces this for HS256)

## Signup & Roles

Users choose their role at signup (`CLIENT` | `SELLER`). Default is `CLIENT` if omitted.

```json
POST /api/auth/signup
{ "name": "Alice", "email": "a@b.com", "password": "secret123", "role": "SELLER" }
```

Validated with `@Valid SignupRequest` — email format, password min 8 chars.

## Kafka Events

ProductService publishes to the `product-events` topic after every mutation.

```json
{
  "eventType": "CREATED|UPDATED|DELETED",
  "productId": "...",
  "sellerId":  "...",
  "name":      "...",
  "price":     0.0,
  "imageIds":  [],
  "timestamp": "2026-05-03T12:00:00"
}
```

- Producer: `ProductEventProducer` — key is `productId` (same partition per product)
- Topic auto-created at startup: 3 partitions, 1 replica (`KafkaConfig`)
- Serializer: `JsonSerializer` (Spring Kafka)

## Resilience — Circuit Breaker

`MediaServiceClient.allExist()` is protected by Resilience4j:

```
@CircuitBreaker(name = "mediaService", fallbackMethod = "allExistFallback")
```

- Fallback throws `ServiceUnavailableException` → HTTP 503
- State visible at: `GET /actuator/health` on ProductService
- Config in `ProductService.yml` (Config Server): sliding window 10, threshold 50%, open 30s
- RestTemplate timeout: connect 3s, read 3s

---

## Distributed Tracing

Every request gets a `X-Correlation-ID` UUID injected by the Gateway's `CorrelationIdFilter`.
Spans are exported to Zipkin: `http://localhost:9411`

To follow a request across services: search by Correlation-ID or trace-id in Zipkin UI.

## Config Server — How It Works

Services fetch config at startup:
```
spring.config.import=optional:configserver:${CONFIG_SERVER_URL:http://localhost:8888}
```

The `optional:` prefix means services start with local `application.properties` as fallback
if Config Server is unavailable. No cascading startup failure.

Config resolution order (highest wins):
1. Environment variables
2. Config Server (service-specific YAML, e.g. `UserService.yml`)
3. Config Server shared (`application.yml`)
4. Local `application.properties`

---

## Service Discovery — How It Works

All services register with Eureka at startup. The Gateway resolves `lb://UserService`
to actual host:port via the registry. No hardcoded URLs needed between services.

Eureka dashboard: `http://localhost:8761`

---

## Inter-Service Calls

- **ProductService → MediaService (sync):** `MediaServiceClient` uses `RestTemplate` with
  circuit breaker. Called internally (bypasses Gateway). MediaService permits all requests
  since the Gateway already validated the JWT upstream.
- **Events (async):** ProductService → Kafka → any future consumer

Do NOT call between services through the Gateway (double routing overhead).

## Code Conventions

- Java: records for DTOs, `@Valid` on request bodies, constructor injection only
- No `@Autowired` on fields — constructor injection always
- No manual `new BCryptPasswordEncoder()` in services — inject the `PasswordEncoder` bean
- Angular: standalone components, typed HttpClient, functional guards, JWT interceptor
- MongoDB: `@Document`, pre-generate `ObjectId` IDs when URL depends on ID (avoids double save)
- Kafka events: fire-and-forget (`kafkaTemplate.send(...)`) — no blocking on ack

## Gotchas

- `spring-boot-starter-web` and `spring-cloud-gateway` conflict — Gateway uses **webflux only**
- jjwt 0.12.6: use `Jwts.parser().verifyWith(key).build()` and `Keys.hmacShaKeyFor(bytes)`
  The old `setSigningKey` / `SignatureAlgorithm` API is removed
- JWT secret must be **≥ 32 bytes** for HS256 — `jjwt-impl` throws `WeakKeyException` otherwise
- MongoDB `ObjectId` — pre-assign with `new ObjectId().toHexString()` when building a URL
  that includes the ID, to avoid saving the document twice
- `spring-cloud-starter-netflix-eureka-client` requires `spring-cloud-dependencies` BOM —
  add `dependencyManagement { imports { mavenBom ... } }` to every service's `build.gradle`
- Config Server `native` profile reads from `classpath:/config` — files must be inside the jar.
  Changing config requires a rebuild unless an external volume is mounted.
- Resilience4j requires `spring-boot-starter-aop` to activate `@CircuitBreaker` proxying
